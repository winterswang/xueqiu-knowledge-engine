# 知识引擎实现 vs SKILL.md 差距分析

## 核心差距总览

| 维度 | SKILL.md 要求 | 当前实现 | 差距 |
|------|--------------|---------|------|
| **数据空间** | ima KB + 笔记双空间 | 本地文件系统 | ❌ 根本性差异 |
| **标签体系** | 4类标签（页面/来源/状态/系统） | 无标签 | ❌ 完全缺失 |
| **命名规范** | `概念/[名]`、`实体/[名]` | 直接文件名 | ❌ 不规范 |
| **索引笔记** | `_知识库索引` + `_操作日志` | 无 | ❌ 缺失 |
| **知识复利** | 每次 ingest 更新所有相关页 | 只更新直接关联页 | ⚠️ 部分实现 |
| **矛盾检测** | `_contradiction_found` 标记 | 无 | ❌ 缺失 |
| **交叉引用** | `[[概念/名]]` wikilink | 无 | ❌ 缺失 |
| **Layer 0** | 概念/实体页直达优先 | 4层但无直达优先 | ⚠️ 检索弱 |
| **batch-ingest** | 扫描→分组→优先级队列→逐批 | 简单批量 | ⚠️ 功能弱 |
| **lint 体检** | 8类检查（KB）+ 5类（笔记） | 只有 validator.py | ⚠️ 覆盖窄 |
| **content-export** | 知识→播客/PPT | 无 | ❌ 未实现 |
| **autoresearch** | 3轮迭代自主研究 | 无 | ❌ 未实现 |

---

## 详细差距拆解

### 1. 数据空间（根本性差异）

**SKILL.md 要求**:
- 操作 ima 的两个独立数据空间（KB + 笔记）
- 所有命令前必须解析作用域
- KB 支持富媒体 + 标签 + 交叉引用

**当前实现**:
- 纯本地文件系统（`knowledge/` 目录）
- 无作用域解析
- 无 ima KB 集成

**影响**: 知识引擎只能在本地运行，无法利用 ima 的全端可用、协作共享、19种格式支持等优势。

---

### 2. 标签体系（完全缺失）

**SKILL.md 要求**:
```
页面标签：_structured_note / _concept_page / _entity_page
系统标签：_raw / _needs_review / _contradiction_found / _orphan / _outdated
来源标签：来源/公众号 / 来源/PDF / 来源/网页
状态标签：状态/待消化 / 状态/已消化 / 状态/待更新
```

**当前实现**:
- 无任何标签系统
- 页面类型通过目录区分（`sources/`、`concepts/`、`entities/`）
- 无法表达来源类型、处理状态、矛盾标记

**影响**: 
- 无法做标签云统计
- 无法标记待 review 的文章
- 无法检测孤儿概念/实体
- lint 检查无法进行

---

### 3. 命名规范（不规范）

**SKILL.md 要求**:
- 来源页：中文描述性标题
- 概念页：`概念/[概念名]`
- 实体页：`实体/[实体名]`

**当前实现**:
- 来源页：`2026-06-30-123456789.md`（日期+ID）
- 概念页：`安全边际.md`（无前缀）
- 实体页：`拼多多.md`（无前缀）

**影响**: 
- 无法一眼识别页面类型
- query 检索时无法优先命中概念/实体页
- 与 SKILL.md 的 Layer 0 直达冲突

---

### 4. 索引笔记（缺失）

**SKILL.md 要求**:
- `_知识库索引`：概念网络区 + 来源笔记区 + 标签云 + 最近更新
- `_操作日志`：每次操作的时间/操作/结果记录

**当前实现**:
- 无索引笔记
- 无操作日志
- 只有 `.git` 作为隐式日志

**影响**:
- 无法快速浏览知识库全貌
- 无法追溯操作历史
- 每次 ingest 后不更新全局索引

---

### 5. 知识网络更新（部分实现）

**SKILL.md 要求**:
- 每次 ingest 后必须更新所有相关的概念页和实体页
- 更新 `_知识库索引` 的概念网络区
- 概念页/实体页只追加不删除

**当前实现**:
- ingest 时更新直接关联的概念页和实体页
- 不更新全局索引
- 概念页 body 被重写（非追加）

**影响**:
- 知识网络生长不完整
- 无法看到知识演变轨迹
- 全局统计缺失

---

### 6. 矛盾检测（缺失）

**SKILL.md 要求**:
- 发现冲突时标记 `_contradiction_found`
- 不自动消解矛盾

**当前实现**:
- 无矛盾检测逻辑
- 新信号直接覆盖或追加

**影响**:
- 同一实体 contradictory 信号无标记
- 用户无法识别观点冲突

---

### 7. 交叉引用（缺失）

**SKILL.md 要求**:
- 笔记内 `[[概念/名]]` 视觉标识
- search 检索支持 wikilink

**当前实现**:
- 无 wikilink
- 概念页中有 `[[entities/拼多多|拼多多]]` 链接，但不完整

**影响**:
- 知识网络的可视化连接弱
- 无法通过点击链接跳转

---

### 8. 检索层级（弱于要求）

**SKILL.md 要求**:
- **Layer 0**: 概念/实体页直达（优先返回已编译的概念页）
- Layer 1: 索引层
- Layer 2: 来源笔记层
- Layer 3: 综合层

**当前实现**:
- Layer 1: 实体直达
- Layer 2: 索引层
- Layer 3: 来源检索
- Layer 4: 综合层
- **无 Layer 0 概念/实体页直达优先**

**影响**:
- 检索效率低于 SKILL.md 设计
- 无法优先返回已编译的知识

---

### 9. batch-ingest（弱于要求）

**SKILL.md 要求**:
- 扫描未消化条目
- 按主题分组
- 优先级队列
- 逐批 ingest

**当前实现**:
- 简单批量处理
- 无主题分组
- 无优先级队列
- 按文件列表顺序处理

**影响**:
- 批量处理效率低
- 无法优先处理高价值文章

---

### 10. lint 体检（覆盖窄）

**SKILL.md 要求**:
- KB 8类检查：孤立页、过期页、空定义、引用断裂、标签缺失、概念冲突、实体不一致、重复来源
- 笔记 5类检查：笔记本分布、未归类、重复、空内容、命名规范

**当前实现**:
- `validator.py`: 空定义、孤立实体、循环引用、时间线排序、标签类型
- 无标签检查（因为没有标签体系）
- 无引用断裂检查
- 无过期检测

**影响**:
- 知识库健康度无法全面评估
- 潜在问题（孤立页、引用断裂）无法发现

---

### 11. 未实现功能

| 功能 | SKILL.md 描述 | 当前状态 |
|------|--------------|---------|
| **content-export** | 知识→播客/PPT | ❌ 未实现 |
| **autoresearch** | 3轮迭代自主研究 | ❌ 未实现 |
| **organize** | 自动归档未归类笔记 | ❌ 未实现 |
| **query(笔记)** | 笔记侧两层检索 | ❌ 未实现 |

## 根因分析

### 为什么有这些差距？

1. **架构选择差异**: 
   - SKILL.md 基于 ima 原生基础设施（KB 笔记 + 标签）
   - 我们选择了本地文件系统（Markdown + YAML + git）
   - 这导致标签、命名规范、索引笔记等 ima 特有功能无法直接映射

2. **范围收缩**:
   - SKILL.md 是通用知识引擎（任意素材）
   - 我们聚焦雪球价值投资文章（特定领域）
   - 因此 content-export、autoresearch 等通用功能未实现

3. **实现顺序**:
   - 先实现了核心管线（extract → ingest → query）
   - 高级功能（lint、batch-ingest、矛盾检测）被后置
